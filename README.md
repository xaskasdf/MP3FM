# MP3FM

Open-source replacement for Sony's `MP3FileManager.exe` -- transfers music to
Sony Network Walkman players that use the OMGAUDIO database format.

Sony shipped these players with a Windows-only transfer utility and never
released a Linux or macOS version.  MP3FM reimplements the transfer protocol
entirely from reverse-engineered documentation, letting you manage your
Walkman from any platform with Python 3.

## Supported Devices

- **NW-E405** (256 MB)
- **NW-E407** (1 GB)
- **NW-E505** (512 MB)
- Other Sony Network Walkman players that store audio under an `OMGAUDIO/`
  directory with `DvID.dat` device identification (NW-E4xx and NW-HD series)

## Features

- Transfer **MP3** files directly (no re-encoding)
- Transfer **FLAC, WAV, OGG, M4A, AAC** with automatic conversion to MP3 via
  ffmpeg
- Configurable MP3 bitrate (default 128 kbps)
- **XOR encryption** of audio data using the device-specific key from
  `DvID.dat`
- Full **OMGAUDIO database generation** (all six DAT index files)
- **EA3/OMA container** creation with ID3-compatible metadata tags
- Preserves ID3 tags: title, artist, album, genre, track number
- Recursive directory scanning -- point it at your music library

## Requirements

- **Python 3.6+**
- **[mutagen](https://mutagen.readthedocs.io/)** -- audio metadata parsing
  ```
  pip install mutagen
  ```
- **[ffmpeg](https://ffmpeg.org/)** -- required only for non-MP3 input
  (FLAC, WAV, OGG, etc.)
  ```
  # Debian/Ubuntu
  sudo apt install ffmpeg

  # Arch
  sudo pacman -S ffmpeg

  # macOS
  brew install ffmpeg
  ```

## Usage

### Basic: Transfer a directory of MP3s

```bash
python3 mp3fm.py /mnt/walkman ~/Music/album/
```

### Transfer individual files

```bash
python3 mp3fm.py /mnt/walkman track1.mp3 track2.flac track3.ogg
```

### Transfer FLAC files at 192 kbps

```bash
python3 mp3fm.py -b 192 /mnt/walkman ~/Music/flac-collection/
```

### Full help

```bash
python3 mp3fm.py --help
```

**Arguments:**

| Argument | Description |
|----------|-------------|
| `device` | Walkman mount point (e.g. `/mnt/walkman`) |
| `files`  | One or more audio files or directories |
| `-b`, `--bitrate` | MP3 encoding bitrate in kbps (default: 128) |

### Mounting the Walkman

The Walkman appears as a USB Mass Storage device.  Mount it normally:

```bash
# Find the device
lsblk

# Mount (typical)
sudo mount /dev/sdb1 /mnt/walkman

# Transfer
python3 mp3fm.py /mnt/walkman ~/Music/

# Sync and unmount
sync
sudo umount /mnt/walkman
```

The device directory structure should look like:

```
/mnt/walkman/
  MP3FM/
    DvID.dat            <-- device encryption key (must exist)
    MP3FileManager.exe  <-- original Sony utility (not needed)
  OMGAUDIO/
    10F00/              <-- OMA audio file directories
    10F01/
    00GTRLST.DAT        <-- database index files (generated by mp3fm)
    01TREE01.DAT
    ...
```

## How It Works

1. **Scan input**: Collects all audio files from the provided paths.  For
   non-MP3 formats, ffmpeg transcodes to MP3 at the requested bitrate.

2. **Read device key**: Extracts the 4-byte encryption key from
   `DvID.dat` (offset 0x0A-0x0D on the device).

3. **Create OMA containers**: For each track, builds an OMA file consisting of:
   - A 3072-byte **EA3 tag header** (ID3v2-compatible, starts with `ea3`
     instead of `ID3`) containing title, artist, album, genre, and
     Sony-specific TXXX frames (`OMG_TRACK`, `OMG_TRLDA`).
   - A 96-byte **EA3 audio header** specifying codec (MP3), bitrate,
     sample rate, channel mode, and the `0xFFFE` encryption marker.
   - The raw MP3 audio data **XOR-encrypted** with a key derived from the
     device ID and track number.

4. **Write OMA files**: Places encrypted `.OMA` files in the
   `OMGAUDIO/10Fxx/` directories using Sony's naming convention
   (`1000XXXX.OMA` where XXXX is the hex track ID).

5. **Generate OMGAUDIO database**: Writes all six DAT index files that the
   player firmware reads at boot:
   - `00GTRLST.DAT` -- master index (tree/group pair references)
   - `01TREExx.DAT` -- tree structures (group and track ordering)
   - `02TREINF.DAT` -- tree information
   - `03GINFxx.DAT` -- group information (album/artist/genre metadata)
   - `04CNTINF.DAT` -- per-track content information
   - `05CIDLST.DAT` -- content ID list

6. **Sync**: Flushes filesystem buffers so the device can be safely ejected.
